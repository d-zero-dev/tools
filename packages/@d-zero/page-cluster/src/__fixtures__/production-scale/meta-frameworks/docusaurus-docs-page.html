設定リファレンス

このページでは、Sample Docsの設定ファイルで指定できる全オプションについて説明します。

備考

設定ファイルを変更した場合は、開発サーバーの再起動が必要です。

基本オプション#

以下は最も基本的な設定項目です。プロジェクトのルートに配置する設定ファイルで指定します。

export default {
  title: 'Sample Docs',
  tagline: 'ドキュメントサイトのサンプル',
  url: 'https://docs.example.com',
  baseUrl: '/',
  organizationName: 'example',
  projectName: 'sample-docs',
};

title#

サイトのタイトルを指定します。ブラウザタブやSEOメタデータに使用されます。

url#

サイトの本番URLを指定します。サイトマップ生成などに利用されます。

プラグインオプション#

プラグインごとに独自の設定を追加できます。以下は検索プラグインの設定例です。

plugins: [
  [
    '@sample/plugin-search',
    {
      indexBlog: false,
      indexDocs: true,
    },
  ],
],
注意

プラグインのバージョンはコア本体のバージョンと合わせる必要があります。

テーマオプション#

ナビゲーションバーやフッターの構成はテーマ設定で行います。

キー型説明
navbar.titlestringナビゲーションバーのタイトル
navbar.itemsarrayナビゲーションリンクの一覧
footer.stylestringフッターのスタイル（darkまたはlight）

高度なオプション#

大規模なドキュメントサイトを構築する際に有用な追加オプションを紹介します。

export default {
  markdown: {
    format: 'mdx',
    mermaid: true,
  },
  future: {
    experimental_faster: true,
  },
};
キー型説明
markdown.formatstringMarkdownの解釈形式
markdown.mermaidbooleanMermaid図の有効化
future.experimental_fasterboolean高速ビルドの実験的機能
備考

実験的機能は将来のバージョンで挙動が変更される可能性があります。

環境変数#

ビルド時に参照される環境変数の一覧です。

  • DOCS_ENV — 実行環境（development / production）
  • DOCS_API_BASE — APIリファレンス生成時のベースURL
  • DOCS_ANALYTICS_ID — アクセス解析用のID
  • DOCS_ALGOLIA_APP_ID — 検索インデックス用のアプリID
  • DOCS_ALGOLIA_API_KEY — 検索インデックス用のAPIキー

よくある質問#

設定に関してよく寄せられる質問をまとめました。

設定変更が反映されない場合は？#

開発サーバーのキャッシュが原因の場合があります。キャッシュをクリアして再起動してください。

複数の設定ファイルを併用できますか？#

環境ごとに設定ファイルを分割し、ビルドスクリプト側で切り替える運用が推奨されます。

MARKER_DOCUSAURUS_DOCS_PAGE
