認証の設定方法

このページでは、CloudFlow APIを利用するための認証設定について説明します。

MARKER_DOCUMENTATION_SITE_WITH_SIDEBAR

概要

CloudFlow APIはAPIキーによるBearer認証を採用しています。すべてのリクエストにはAuthorizationヘッダーが必要です。

Note

APIキーは組織の管理画面から発行できます。発行後の再表示はできないため安全に保管してください。

APIキーの発行

管理画面の「設定」→「APIキー」から新規キーを発行します。

手順

  1. 管理画面にログインする
  2. 「設定」メニューを開く
  3. 「APIキー」タブを選択する
  4. 「新規発行」ボタンをクリックする

リクエスト例

curl -X GET https://api.cloudflow.example/v1/tasks \
  -H "Authorization: Bearer YOUR_API_KEY"

トークンの有効期限

発行されたAPIキーはデフォルトで無期限です。セキュリティポリシーにより有効期限を設定することも可能です。

Warning

APIキーをクライアントサイドのコードに直接埋め込まないでください。漏洩のリスクがあります。

レスポンス例

{
  "id": "task_123",
  "title": "サンプルタスク",
  "status": "open"
}

OAuth 2.0によるユーザー認証

サードパーティアプリケーションからのアクセスにはOAuth 2.0フローを利用してください。

認可コードの取得

GET https://auth.cloudflow.example/oauth/authorize?
  client_id=CLIENT_ID&
  redirect_uri=REDIRECT_URI&
  response_type=code

次のステップ

認証が完了したら、タスクの作成に進んでください。
